üìö Git Basic Concepts

Understanding the Four Areas:

1. Working Directory: Where you edit files - untracked changes

2. Staging Area: Prepared changes ready for commit

3. Local Repository: Your computer's Git history

4. Remote Repository: GitHub server for collaboration

git add filename.txt # Move to staging
git commit -m "message" # Save to local
git push origin main # Upload to GitHub
git pull origin main # Download updates

üîÑ Git Workflow

Complete Git Workflow:

Step 1: Initialize - Create Git repository

Step 2: Edit - Make changes to files

Step 3: Stage - Select changes to commit

Step 4: Commit - Save snapshot

Step 5: Push - Upload to GitHub

Step 6: Pull - Get latest updates

git init # Start new repo
git add . # Stage all files
git commit -m "First commit" # Commit
git remote add origin URL # Connect to GitHub
git push -u origin main # Push
git pull origin main # Pull updates
üí° Tip: Commit often! Small, frequent commits are better than large ones.

üåø Branching Strategy

Why Use Branches?

Isolation: Work on features without affecting main code

Parallel Development: Multiple features at same time

Safe Experiments: Try new ideas safely

Team Collaboration: Everyone works on their own branch

git branch feature-name # Create branch
git checkout feature-name # Switch branch
git checkout -b new-branch # Create & switch

git branch -d feature-name # Delete branch

üö® Troubleshooting

Common Problems & Solutions:

Problem: "I committed to wrong branch!"
git log # Copy commit hash
git checkout correct-branch
git cherry-pick <hash>
git checkout wrong-branch
git reset --hard HEAD~1
Problem: "Need to undo last commit!"
# Keep changes
git reset --soft HEAD~1

# Discard changes
git reset --hard HEAD~1

# Already pushed? Use revert
git revert HEAD
Problem: "Merge conflict!"
git pull # Shows conflict
# Open file, find <<<, ===, >>> markers
# Edit file, keep what you want
# Remove markers
git add file.txt
git commit -m "Resolve conflict"
git push
Problem: "Deleted file by mistake!"
# If not committed yet
git checkout -- filename.txt

# If already committed
git log --all -- filename.txt
git checkout <hash> -- filename.txt

üíº Interview Questions

Top Git Interview Questions:

Q1: What is Git?

Git is a distributed version control system for tracking changes in source code. It allows multiple developers to work together, maintains complete history, and enables branching/merging.

Q2: Git vs GitHub?

Git is the tool (software on your PC). GitHub is the platform (cloud hosting service). Like Microsoft Word vs Google Docs.

Q3: git pull vs git fetch?

fetch: Download changes only, don't merge
pull: Download AND merge automatically (fetch + merge)

Q4: What is a merge conflict?

When two branches modify same part of file differently. Git can't auto-decide. You must manually choose which changes to keep.

Q5: git merge vs git rebase?

merge: Creates merge commit, preserves history
rebase: Linear history, cleaner but rewrites commits
‚ö†Ô∏è Never rebase shared branches!

Q6: What is HEAD?

Pointer to current branch reference. Represents "where you are" in repository history.

Q7: Explain git stash

Temporarily saves uncommitted changes so you can switch branches. Reapply them later with git stash pop.

Q8: What is .gitignore?

File specifying which files/folders Git should ignore. Used for dependencies, build files, sensitive data.
